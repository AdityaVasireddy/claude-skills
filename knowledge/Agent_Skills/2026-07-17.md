## Cases

### CASE: write-before-gate-reorder
status: auto
problem: `run_pipeline()` created the `knowledge/` vault (mkdir, seeded `.gitignore`, seeded `capture-rules.md`, `sweep.log` append) before checking transcript existence or the triviality gate, so every session end — even ones that immediately SKIP — stamped a vault into whatever cwd it ran in, confirmed live in headless `-p` children and known to have polluted an unrelated project's repos.
decision: moved all filesystem writes to `$VAULT` below the transcript-existence check and triviality gate; SKIP events now log to a new global `~/.claude/historian/skips.log` instead of a per-project vault.
why: desired behavior is that a session that would SKIP leaves the cwd absolutely untouched — zero files, zero dirs — which is only possible if skip logging never targets the session cwd.
tags: [bugfix, hooks, historian, filesystem]

### CASE: sessionend-precompact-double-fire-lock
status: auto
problem: SessionEnd and PreCompact both run the same sweep pipeline; the existing byte-growth dedupe (keyed on session ID in `knowledge/.sweep/sessions.log`) only guards against a prior *completed* sweep and does nothing if both hooks are already running concurrently before either has recorded its bytes.
decision: added an atomic `mkdir`-lock keyed on session ID under `~/.claude/historian/locks/`, with stale-lock reclaim and an empty-`SESSION_ID` guard, checked before the model call.
why: the model call is the expensive step (real token cost) — the byte-based dedupe can't catch the concurrent-firing window, so the lock has to sit earlier in the pipeline than that check.
tags: [concurrency, hooks, dedupe, historian]

### CASE: historian-sweep-active-frozen-interface
status: auto
problem: `HISTORIAN_SWEEP_ACTIVE` (first line of the script, the recursion guard) is now relied on by an external system, not just internally.
decision: kept the guard exactly as-is — never renamed, nothing may run before that check — and documented it inline as a frozen interface.
why: issue-runtime's ADR-22 option B sets this var in an engine child's env specifically to suppress the sweep; that dependency isn't visible from this repo alone, so it had to be called out explicitly to avoid future breakage.
tags: [interface, historian, hooks, compatibility]

### CASE: regression-coverage-scope-skip-redundant-live-test
status: auto
problem: after the fix, whether to also drive a fresh live multi-turn chat session purely to re-prove that normal (non-skip) sweeps still work, on top of the automated suite.
decision: relied on the existing automated regression suite (T2–T4, T8, T15–T17, T19–T20, all asserting real sweep/day-file/commit behavior, 75/75 passing) as sufficient evidence instead of running a redundant live session.
why: judged the automated coverage exhaustive enough that an extra live multi-turn test would be redundant re-proof of already-asserted behavior.
tags: [testing, historian, judgment-call]
