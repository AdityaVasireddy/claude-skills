## Cases

### CASE: cp-preserve-mode-over-chmod-in-compound-sync-command
status: auto
problem: Syncing the updated skill files to `~/.claude/skills/` and `~/.claude/historian/` kept getting denied by the permission system, even after the user explicitly authorized the deployment task.
decision: Drop `chmod +x` from the sync command entirely and rely on `cp --preserve=mode` (the source file already has the execute bit), instead of continuing to retry the chmod-bearing compound command.
why: Inspection of `~/.claude/settings.json` showed a permission deny rule `"Bash(chmod *)"` — the bundled `chmod +x` calls inside the larger compound command triggered a blanket denial of the whole command, not a directory-write restriction as first assumed after three failed retries.
tags: [permissions, deployment, bash, sync]

### CASE: grep-c-empty-match-fallback-doubles-output
status: auto
problem: The newly-built `--doctor`/`--status` diagnostics rendered the pending-queue line oddly on a truly-empty queue.
decision: Stop using `$(grep -c . "$f" 2>/dev/null || echo 0)` for the pending count in both `doctor_mode` and `status_mode`; the existing `|| true` pattern used elsewhere in the script is safe and should be used instead.
why: `grep -c .` exits nonzero on zero matches even though it has already printed a valid `0`, so the `|| echo 0` fallback appends a second `0` — a bug in the diagnostics code just written, not a pre-existing issue, and it slipped past the 62/62 test suite because no existing test exercised `--doctor`/`--status` against a truly-empty `pending.log`.
tags: [bash, diagnostics, bug-fix, testing-gap]

### CASE: skip-live-replay-verify-via-existing-regression-test
status: auto
problem: Task 8 required verifying `--replay` works correctly, but both live vaults (`Agent_Skills` and `issue-runtime`) had 0 pending entries, so there was nothing real to replay.
decision: Verify only the `--replay` error/usage path live, and rely on the existing regression test T19 (which queues a failure, replays it, confirms the day file and pending-log removal, then replays again for idempotency) to cover the success path instead of manufacturing a live run.
why: Running `--replay` against an empty queue would exercise nothing and would just be a wasted model call; T19 already exercises the full success/idempotency behavior end-to-end.
tags: [verification, testing, replay, deployment]
